Bliss

A glossy, optimistic interface language — friendly chrome you can see and want to press. A respectful, modern homage to the Windows XP "Luna" visual style, re-tokenized so you can build with it today.

Whistler 2000–2001 › Luna 2001 › the three schemes: Blue, Olive Green, Silver. One spine: computing should feel welcome.

Getting started

Two lines to adopt the system. It's namespaced (--bl-* / .bl-*), ships no global reset, and styles no bare elements — so it drops into an existing project without collisions.

1 · Link the stylesheet (and the optional behavior layer).   2 · Add .bl-root to opt a region into the Bliss surface.   3 · Set data-bl-theme and use components.

Want only the design tokens for your own components? You don't even need .bl-root — the --bl-* custom properties live on :root and are available everywhere. Tooling consumers can read bliss.tokens.json. Bliss uses the real XP faces (Trebuchet MS / Tahoma), which ship on Windows; on other platforms it falls back to system-ui, so there's nothing to load.

Color

The chrome is one glossy gradient, and its hue is a swappable scheme: the same title bar becomes Blue, Olive, or Silver by changing a handful of tokens. Reference the semantic roles in product code — they retint with the theme. Click any swatch to copy its token.

The three Luna schemes

Color is a setting, not a rebuild. The Bliss field (sky + hill) stays constant; the chrome takes the user's color. Switch live with the toggle in the top bar — the whole page retints from one attribute.

Build with the gloss

Buttons

The classic raised Luna 3D face. The default carries the accent edge; the CTA wears the green "go" orb. Full state coverage, three sizes.

Surfaces & cards

Bright, roomy surfaces on the warm Luna face. .bl-panel is the bare surface; .bl-card adds a header and padded body. The .bl-bliss helper paints the constant sky-and-hill field.

Forms & fields

Sunken, square-cornered Luna inputs — the authentic blue-grey field border, an inset shadow, and an accent focus ring. Inputs, selects, search, and a numeric spinner wired by bliss.js.

Choices

Checkbox, radio, switch, and trackbar — each built on ARIA roles and made live by bliss.js: keyboard-operable, with the candy-pop check animation and a sliding switch knob.

Tabs & groups

The classic Luna property sheet: the active tab lifts onto the page face while the others sit a step back. Tabs are roving-tabindex and arrow-key navigable; the group box is a labelled fieldset.

Status & feedback

Badges always pair a colored dot with a label, so meaning survives any color-vision difference. Plus progress, a list view, the XP-yellow tooltip and balloon, and glossy toasts.

Window chrome

The full Luna window: a glossy themed title bar with min/restore/close buttons, a menu bar with real dropdowns, a toolbar, and a status bar. Inactive windows desaturate, exactly as XP did.

Dialogs

An overlay dialog with the same themed title bar. Open it with a data-bl-open hook (or Bliss.openDialog); close it with the backdrop, a data-bl-close button, or Esc.

Desktop shell

Bliss's whole thesis in one frame. A window manager activates automatically for any .bl-desktop: double-click an icon (or use the Start menu) to open a draggable window, focus brings it forward and dims the rest, and the taskbar tracks what's open. The clock ticks live.

System information

A diegetic "System Properties" dialog — built from the same chrome, with live values that reflect the scheme you've chosen on this page.

Principles

Six rules, each lifted from something Luna actually did — then stated as a rule you can hold a design against.

01

Optimism by default

Light, sky, and air — the screen should feel like a good morning.

Bliss starts bright, not dark. The interface greets you instead of bracing you.

02

Gloss is the material

A highlight up top, a saturated body below — light reads as a curved surface.

The signature gradient turns flat color into something that looks lit and pressable.

03

Soft and tactile

Rounded corners, raised bevels, controls that look like buttons.

Everything invites the hand. A button is unmistakably a button.

04

Color is a scheme

One accent themes the whole chrome. Swap it without rebuilding.

Blue, Olive Green, Silver: the same Windows, reskinned by a setting.

05

Guide the way

Task panes, plain words, and a Start button labelled "start."

The interface names where to begin, so nobody has to already know.

06

Confident chrome

The frame is allowed to be seen, and to be cheerful about it.

A glossy title bar and a green orb are landmarks, not noise.

Do, and do not

Do

  • Build every bar and button from the gloss layers: highlight, step, body, base shadow.
  • Keep one accent scheme driving all the chrome, and let users switch it.
  • Stay bright and roomy — light surfaces, open sky, generous padding.
  • Make controls obviously pressable with rounded corners and raised bevels.
  • Name the way forward: a labelled Start, task panes, plain-language headings.
  • Keep motion short and gentle, with a little pop where it earns a smile.

Do not

  • Flatten the chrome to a solid fill — without the gloss it isn't Luna, it's a rectangle.
  • Mix three accents on one screen; the scheme is one color, themed everywhere.
  • Go dark-and-moody — that betrays the optimism the whole language is built on.
  • Ship ambiguous, frameless controls that hide whether they can be pressed.
  • Assume expertise; never strip the guidance that makes Bliss welcoming.
  • Over-animate. Bouncing everything turns friendly into frantic.

Accessibility

Friendly should mean reachable. XP shipped High Contrast themes and a magnifier; Bliss carries that intent — color is never the only signal, motion is optional, and every control works from the keyboard.

Built in

Color is never alone — status pairs a colored dot with a text label, and selections carry the authentic dotted XP focus rectangle (light on chrome, dark on the face). Checkboxes, radios, tabs, switches and windows respond to Space/Enter/arrows; Esc closes menus, dialogs and the Start menu. All motion collapses under prefers-reduced-motion, and a @media print block hides the taskbar, menus and toasts.

Your part

Lists, tabs and windows are styled, not re-implemented — keep correct roles and aria-*. Reserve the accent for emphasis and chrome; use --bl-ink / --bl-ink-dim for prose, and don't place body text on a saturated title-bar fill. The chime never plays without a user gesture.

Heritage

For years the desktop was gray, beveled, and businesslike. Then Whistler shipped as Windows XP with the Luna visual style — rounded windows, glossy gradient title bars, a green Start button, and a photograph of a hill under a blue sky. Critics called it "Fisher-Price." Hundreds of millions called it home.

1995–2000
Classic
Win95 through 2000. Square grey bevels, raised and sunken edges. Capable, sober, never warm.
2001
Whistler → Luna
XP ships the themeable visual-styles engine on Oct 25, 2001. Rounded, glossy, green Start, ClearType text.
2001–2006
Three schemes
Blue, Olive Green, and Silver reskin the same Windows — color as a setting, not a rebuild.
2006→
Aero & after
Gloss gave way to glass, then to flat. The friendliness, quietly, was the part worth keeping.
Now
Bliss
Luna's optimism, re-tokenized: glossy, tactile chrome you can build with again.
